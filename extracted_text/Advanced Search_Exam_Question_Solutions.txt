SHORT ANSWER
================================================================================

1. When should you use optimization techniques (like hill climbing) instead of path-finding search (like A*)?

   Use optimization techniques when:
   - There is no explicit start and goal state; you seek the best configuration in a solution space.
   - The problem is about finding a state that maximizes/minimizes an objective function f(x) (e.g., scheduling, parameter tuning, TSP).
   Use path-finding search when:
   - There is a clear start state and goal state, and you need to find a path (sequence of actions) from start to goal.
   - Solution quality is measured by path cost (e.g., navigation, puzzle solving).

2. What is the "local optima problem" in hill climbing? Why does it occur?

   The local optima problem is when hill climbing gets stuck at a state that is better than its immediate neighbors but worse than the global optimum. It occurs because hill climbing only moves to a neighbor if it improves f(x); once no neighbor improves f(x), the algorithm terminates. There is no mechanism to escape a local optimum by temporarily accepting a worse state.

3. Explain the exploration versus exploitation tradeoff in simulated annealing. How does temperature control this tradeoff?

   Exploration: trying new regions of the search space (often by accepting worse moves). Exploitation: moving toward better states.
   - High T: acceptance probability for worse moves is higher → more exploration (random walk–like behavior).
   - Low T: acceptance probability is lower → more exploitation (behaves like hill climbing).
   Temperature controls this: as T decreases, the algorithm shifts from exploration to exploitation.

4. What is the purpose of the cooling schedule in simulated annealing? What happens if the temperature decreases too quickly?

   The cooling schedule controls how temperature T decreases over time. Its purpose is to allow enough exploration early on (high T) and then converge to a good solution (low T).
   If T decreases too quickly: the algorithm may settle into a local optimum before escaping, behaving almost like hill climbing. Not enough exploration occurs.

5. In genetic algorithms, what is the role of crossover? How does it differ from mutation?

   Crossover: combines genetic material from two parent solutions to produce offspring. It recombines good building blocks (subsolutions) from different individuals.
   Mutation: randomly modifies genes in a single individual. It introduces new diversity and helps escape local optima.
   Crossover exploits existing good solutions; mutation explores new regions.

6. Explain why elitism is commonly used in genetic algorithms. What problem does it solve?

   Elitism keeps the best individual(s) from the current generation in the next generation without modification.
   It solves the problem of losing the best solution when crossover and mutation might degrade it. Without elitism, a very fit individual could be lost and the population might drift away from good solutions.

7. How does the definition of "neighbor" affect hill climbing performance? What is the tradeoff between small and large neighborhoods?

   Small neighborhood: fewer options per step, may miss good moves, faster per step, but more likely to get stuck in poor local optima.
   Large neighborhood: more options per step, better chance to escape local optima and find good solutions, but each step is slower (more evaluations).
   Tradeoff: larger neighborhoods improve solution quality but increase computational cost per iteration.

8. Why might a genetic algorithm outperform hill climbing on a problem with many local optima?

   Genetic algorithms maintain a population of solutions exploring multiple regions simultaneously, can escape local optima via mutation and crossover, and do not rely on a single current state. Hill climbing, with one current state, easily gets stuck in a local optimum and has no mechanism to escape.

================================================================================
TRUE OR FALSE
================================================================================

1. A* search is more memory-efficient than hill climbing and genetic algorithms.
   FALSE. A* stores the frontier and explored set; it can use O(b^d) memory. Hill climbing and GAs typically use much less memory (hill climbing: current state and neighbors; GAs: fixed population size).

2. Hill climbing is guaranteed to find the global optimum if the search space has only one local optimum.
   TRUE. If there is only one local optimum, it must be the global optimum. Hill climbing will terminate there.

3. In simulated annealing, decreasing the temperature too quickly can cause the algorithm to behave like hill climbing.
   TRUE. Quick cooling reduces the probability of accepting worse moves, so the algorithm soon only accepts improving moves, like hill climbing.

4. Genetic algorithms maintain a population of solutions, which helps them explore multiple regions of the search space simultaneously.
   TRUE. Multiple solutions in different regions allow parallel exploration.

5. In the 2-opt neighborhood for TSP, reversing a segment of the tour always produces a valid tour.
   TRUE. 2-opt reverses a contiguous segment; the result remains a valid Hamiltonian cycle (tour).

6. A larger neighborhood in hill climbing always leads to better solutions because more options are considered.
   FALSE. Larger neighborhoods can improve solutions but do not guarantee better results; they increase per-step cost and may still lead to local optima.

7. In genetic algorithms, a higher mutation rate always leads to faster convergence to the optimal solution.
   FALSE. Too high mutation disrupts good solutions and can prevent convergence; there is an optimal range.

8. Random restart hill climbing can find the global optimum with probability approaching 1 if run with enough restarts.
   TRUE. With enough independent random restarts, the probability of at least one restart finding the global optimum approaches 1.

2. Simulated annealing: P = e^(-|f(current) - f(neighbor)|/T), maximizing f.

   (a) f(current)=50, f(neighbor)=45, T=10:
       P = e^(-|50-45|/10) = e^(-5/10) = e^(-0.5) ≈ 0.6065

   (b) Same, T=2:
       P = e^(-5/2) = e^(-2.5) ≈ 0.0821

   (c) As T → 0: P for worse moves → 0. Only improving moves accepted → behaves like hill climbing.
       As T → ∞: P → 1. All moves accepted → behaves like random walk (pure exploration).

3. Algorithm recommendations:

   (a) Hill Climbing. Smooth, predictable landscape; good starting point; fast evaluations. Local search is efficient.

   (b) Genetic Algorithm or Simulated Annealing. Huge combinatorial space; need good feasible solution, not necessarily optimal. GA can handle complex constraints; SA can escape local optima.

   (c) Simulated Annealing. Rugged landscape; need to accept worse moves to reach better regions. SA is designed for this.

   (d) Hill Climbing. Single smooth peak; start near top. Simple hill climbing is sufficient and efficient.
